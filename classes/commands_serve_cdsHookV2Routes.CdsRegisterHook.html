Class CdsRegisterHook

CdsRegisterHook - registers a cds-hook with the federated server

POST /cds-register-hook/:hook-id

The hook-id must be unique across the federation.

In addition to the required definition, the registering service must pass in it's service-name and port as the address field

e.g.

{
	"address": "my-elixir-ai-research-study-matching-program:8080",
	"definition": {
		"hook": "research-study",
		"name": "My Elixir AI Research Study CDS Hook",
		"description": "The research study hook",
		"id": "my-elixir-cds-service-research-study",
		"prefetch": { "patient": "Patient/{{context.patientId}}" }
	}
}

To use a particular cds-hook the client must POST the expected payload to /cds-services/:hook where :hook is obtained from the definition (e.g. "research-study" in this example).

The client must execute any prefetch calls first and send any Bundle resources with the context field. NOTE: the context field is described by the specification but not shown here.
