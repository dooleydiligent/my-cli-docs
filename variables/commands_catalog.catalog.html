Publish a My Elixir Catalog manifest

See also commands/cds-hook.deploy

e.g.

my-cli catalog publish /path/to/folder/with/catalog/manifest.json

A My Elixir Catalog is a collection of resources used to capture consumer input. The example below is a rendition of the FINDRISC Diabetes Mellitus Risk Questionnaire, which is used by clinicians to predict the likelihood of a patient developing diabetes within the next decade.

The catalog is published from a folder with the following structure:

  • manifest.json
  • package.json (must include a build script, which is executed before publication to transpile the source into executable code)
  • favicon.ico (required for publication)
  • tsconfig.json
  • src/index.ts (or other typescript code)

The following required fhir resources are published as part of a My Elixir Catalog:

Organization - Composed of the unique provider name and the unique catalog name, e.g. 'my-elixir' and 'findrisc' in this example

Questionnaire - The form used to collect the input

Bot - The typescript code which captures the input from the Questionnaire

Subscription - The trigger which links the Questionnaire to the Bot

In addition to these resources, an image is required for each published CatalogEntry. These images should be unique.

When a consumer completes the questionnaire, the results are evaluated by the Bot. Any messages generated by the Bot can be sent back to the consumer by creating Communication resources. The referenced Organization is the originator of any such communication. By default these communications will go to the consumer feed.

IMPORTANT The publication of the Bot entails the use of npm install and npm run build
