my-cli

Command line maintenance and provisioning utility for my-elixir.

One of the core components of the "agent" image which is the base of the meapi container.

about the name

Originally my-cli was named "my-elixir" but that is just too long to type. my-cli is intended as a developer/operations support analyst tool so it must be easy to use. my-cli was chosen because it almost sounds like "my-elixir" and is easier to type.

build

npm run build

install

npm pack
npm install -g my-elixir-cli-<version>.tgz

Usage

my-cli --help

Usage: myelixir [options] [command]

- Command to provision My-Elixir -

Options:
  -V, --version                            output the version number
  -h, --help                               display help for command

Commands:
  message [options]                        Send a message to a consumer customer
  migrate                                  Update My Elixir AI database schema
  search [options] <resourceType> <query>  Search for fhir resources
  catalog [options]                        Publish or Unpublish a catalog manifest
  cds-hook [options]                       Deploy, Undeploy, or Serve a cds-hook
  serve [options]                          Serve the my elixir api
  help [command]                           display help for command

message

Send a message to a user. Messages tagged with 'feed' are intended to be part of the customer feed.

Options

  • --toKind - The recipient resource type (default Patient)
  • --to-id - <to-id> The recipient id
  • --message - <message> The required message
  • --tag - The message tag(s) (default 'feed')

Used by test-data-bot to send a message when the consumer record is modified by script

Example:

my-cli message --client-id ${CLIENT_ID} --client-secret ${CLIENT_SECRET} --base-url ${BASE_URL} \
      --to-id ${TESTID} --message "Your history has been modified by import"

export const message = createMeCommand('message') .option('--toKind', 'The recipient resource type (default Patient)', 'Patient') .option('--to-id ', 'The recipient id') .option('--message ', 'The required message') .option('--tag', 'The message tag(s) (default "feed")', 'feed')

migrate - Database migrations

Updates the database with migrations supplied in code.

my-cli migrate update -s <MeduplumDatabaseSecret> --region us-east-1

search

Options

  • --resourceType - The fhir resource (e.g. Patient)
  • --query - The search query

Usage: Find a resource or resources

Used by test-data-bot to locate a user

Example:

my-cli search --client-id ${CLIENT_ID} --client-secret ${CLIENT_SECRET} --base-url ${BASE_URL} Patient email=${TOUSERID}
